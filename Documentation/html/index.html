Welcome to the Structure SDK (Cross-Platform) SDK for Linux!

This release is a starter package that can stream live data from a Structure Core sensor and record/replay .occ capture files.

Prerequisites

The Structure SDK (Cross-Platform) SDK is built and tested on Ubuntu 16.04, but should run on other distributions provided the dependencies below are available.

  • CMake 3.8+
  • GNU make 4.0+
  • gcc/g++ 5.3+
  • libusb 1.0+
  • OpenGL implementation providing libGL and libEGL
  • X11 libraries: libxcursor, libxinerama, libxrandr

The following packages may be installed on Ubuntu 16.04:

  • libc6-dev
  • libstdc++-5-dev
  • libgl1-mesa-dev
  • libegl1-mesa-dev
  • libusb-1.0-0-dev
  • libxcursor-dev
  • libxinerama-dev
  • libxrandr-dev

Getting Started

Grant non-root users access to Structure Core devices by running sudo Driver/install-structure-core-linux.sh.

To build the sample apps, run Scripts/build.sh. Output files will be placed in a subdirectory of Builds/ depending on build configuration and target architecture. Pass -h or --help to see available options.

Recorder Sample App

$ Builds/<config>-<arch>/Samples/Recorder/Recorder

The Recorder app demonstrates streaming and visualization of camera and IMU data from a Structure Core sensor. Basic streaming configuration can be changed using on-screen controls. Run Recorder -h to see additional options that can be specified on the command line, including headless (non-graphical) operation and OCC recording/playback.
